# algorithm_lv4

알고리즘 학습을 **총 11번의 수업**으로 진행하며, 그 일정을 **11주 동안(주당 1회 수업)** 맞춘 저장소입니다.  
디렉터리 이름의 `week1`, `week2`, …는 위 11회 중 **몇 번째 주·몇 번째 회차**에 해당하는 학습 묶음을 뜻하고, **`week11`은 열한 번째 주(마지막 회차)** 를 가리킵니다.

모든 주차를 길게 설명하기보다, 각 주차의 핵심 주제와 학습 산출물을 간단히 기록하는 것을 목표로 합니다.  
「주차별 개요」에는 **지금 저장소에 포함된 주차만** 적습니다. 새 주차 디렉터리가 커밋되면 그때 같은 형식으로 항목을 늘립니다.

```text
│  ├─ main.ipynb

└─ README.md
```

## 주차별 개요 (요약형)

### 1·2주차 — 우선순위 큐

- `week1`: `heapq` 기반 최소 힙 연습(삽입·추출·결과 누적), 절댓값 기준 우선순위 처리 응용
- `week2`: 우선순위 큐 응용 — `숫자 카드 정렬 게임`(묶음 합치기 최소 비용), `쇼핑몰`(계산대 배정·퇴장 시뮬레이션, 이중 힙)

### 3·4주차 — 플로이드-워셜

- `week3`: `경로 찾기`(BOJ 11403, 도달 가능성), `플로이드`(BOJ 11404, 모든 쌍 최단거리), `저울`(KOI 2014, 비교 관계 전파)
- `week4`: `마을 왕복`(BOJ 1956, 최소 사이클), `키 순서`(KOI 2011, 간접 비교로 순위 확정 가능 인원)

### 5주차 — 다익스트라

- `week5`: 양의 가중치 그래프에서 한 쌍 최단거리 — `최소 비용 경로`(BOJ 1916 유형, 단방향, 인접 리스트·O(V²) 다익스트라), `1번에서 N번까지, 두 필수 경유지`(BOJ 1504 유형, 양방향, 다익스트라 3회로 두 방문 순서 비교)

### 6주차 — 우선순위 큐 다익스트라

- `week6`: 대규모 V·E에 대응하는 **최소 힙 다익스트라** — `디랩이의 모든 도시까지 이동 계획`(BOJ 1753 유형, 시작점에서 모든 정점까지 거리), `최소 이동 비용과 경로`(BOJ 11779 유형, 한 쌍 최단거리·경로 복원). 상세는 `week6/README.md` 참고.

## 노트북 구성

`week1`~`week4`의 `main.ipynb`는 문제마다 아래 학습 흐름으로 구성됩니다.

1. 문제 설명
2. TODO 코드(학습자용)
3. TODO 주석 포함 정답 코드
4. 간결 정답 코드(제출용)
5. Mermaid 다이어그램(핵심 아이디어/흐름도/예제 추적)

`week5/main.ipynb`는 TODO 단계 없이 **문제 설명 → 다익스트라 구현 코드 → 핵심 아이디어·알고리즘 흐름도·예제 추적(Mermaid)** 순으로 정리합니다.

`week6/main.ipynb`는 `week1`~`week4`와 같이 문제마다 **TODO → 정답 → 간결 코드 → Mermaid** 흐름을 유지합니다.

## 실행 방법

- 파이썬 스크립트 실행: `python3 <파일명>.py` (주차별 `main.py`는 보조용이며, 일부 주차는 비어 있을 수 있음)
- 주피터 노트북 실행 예: `jupyter notebook week5/main.ipynb`
